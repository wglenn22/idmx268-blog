As web developers regardless of skill level we all know the beginnings of HTML and CSS. This time around let’s explore some of the history of animation and webpages which will kick off a discussion of CSS animations, CSS transitions, and the pros and cons of each.

Many agree that early websites were…well just plain boring, but 1995 was a good year for websites, Javascript was invented and a beautiful piece of web design history was created. The Batman Forever website which was one of the first few websites to use animations albeit via GIF was created. GIF or Graphic Interchange Format was created in 1987 by ComputerServe and dominated the web design field alongside Javascript for the next 10 years.

Things in web animation changed again in 1996 with the creation of Flash by Adobe. Flash created a big boom in web design animation with websites on the internet filled with animation, interactivity, and video. You might be thinking “if flash was so great why is it no longer used?”. You can thank the folks over at Apple for Flash’s early retirement. With the advent of mobile devices Flash’s hit on the CPU made it a no-go for mobile devices to run it as it drained the battery too fast. So, Apple decided not to include any flash support on any of its mobile devices to help maintain battery performance. And just like that he’s(flash) dead Jim. Developers removed and no longer wrote flash code and Flash became like an unloved stepchild. During this period the internet entered into a semi-animated dark age. That is until the introduction of CSS3(&HTML5). Sure, Javascript was around the entire time but developers who weren’t well versed in Javascript needed Javascript programmers to complete the animated parts of their websites. Ideally a good developer can make use of HTML, CSS, Javascript, and our old friend SVG but CSS3 made animations and transitions that rely solely on it and HTML possible.

Transitions and Animations are very similar and are very easily confused, but both are simple and easy to learn. Transitions are even more simplistic when come compared to animations. They are connected to a triggering action, like a mouse click, which causes an object to alternate between two states. There are 5 properties used to create transitions. Transition-property, Transition-duration, Transition-timing-function, transition-delay, transition.

You can use CSS transitions to change an elements color, scale, or position. As well as rotate, skew, and translate objects.

Animations are more complicated and more versatile. With CSS animations you can set different stage that alter behavior of the element multiple times during its cycle. Key frames are used to specify the values for the animation you can do this through the @keyframes at-rule. You can have as many key frames specifications as you want, but 3 things after must be included: the keyframe name, keyframe back, and the key frame selector. This allows the creation of animations that rival that of our beloved unloved stepchild (Flash) we don’t talk about anymore.

Some Pros relating to using both CSS transitions and animations are that you can visualize property changes. Then you can set functions to control the rate at which their property values change. Some Pros relating only to CSS transitions are once again the simplicity because you can only animate between the start and end state. CSS transitions are also more smoothly integrated with Javascript. Some cons are that the property can sometimes be too simplistic for certain interactions.

Some pros of CSS animations are that you can loop them endlessly and it doesn’t require a triggering action. Some cons are that not all properties of SVG can be animated with CSS and require the Use of Javascript.
